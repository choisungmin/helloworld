0. COMMON

* 작업전 준비사항

  • html , css ,js의 주석으로 영역표시를 확실히 한다.
  • 시맨틱 마크업을 중점으로 코딩 준비를 한다.
  • 줄바꿈으로 컨텐츠 영역을 구분을 확실히 한다.
  • 웹폰트 사용여부를 반드시 확인한다.
  • directory > html > css > javascript 순으로 공통 확인후 작업시작한다.

* 작업중 체크사항

  • a태그에 title 속성을 확인한다.
  • images태그에 alt 속성을 확인한다.
  • table 태그사용시 caption 태그를 같이 사용했는지 반드시 확인한다.
  • fieldset 태그 사용시 legend 태그를 같이 사용했는지 확인한다.

1. HTML

1-1. 기본

  • W3C(World Wide Web Consortium)의 내용에 맞게 시맨틱 마크업을 지향하여 코드를 작성한다. (https://www.w3.org)

  • 웹 접근성 지침 가이드에 맞게 논리적인 컨테츠로 구성한다.(http://www.wah.or.kr)

1-3. 네이밍

1-3-1. 규칙

A. 일반

  • 이름은 영문 소문자 , 숫자 조합으로 언더바(_)로 구분하여 작성한다.
  • 이름의 시작단어를 숫자로 시작하거나 대문자로 작성하지 않는다.
네이밍 규칙 예제 테이블
잘못된 예 올바른 예
01Iconnotice icon_notice_01
NoticeIconBtn btn_notice_icon

B. 예약어

  • 예약어가 있는 경우 예약어를 사용한다.
  • 예약어는 공통으로 사용하는 네이밍 예약어에 근거하여 작성한다.
  • 예약어가 없으면 해당 컨텐츠의 의미에 맞게 작성한다.

C. 조합

  • 영문 소문자 , 언더바(_)를 이용하여 작성한다.
  • 프로젝트 작업 가이드에 따라 하이픈(-)을 사용할 수도 있다.
  • 언더바(_)를 사용할 시에는 2개이상의 단어를 조합할 때만 사용한다.
  • 숫자를 이용할 시에는 zero base로 작성한다.
네이밍 규칙 예제 테이블
잘못된 예 올바른 예
brandData brand_data
brand1 brand_01

1-3-2. ID & CLASS

  • ID값은 고유 식별자 이므로 한페이지당 1개씩 사용한다.(중첩 불가능)
  • ID값은 주로 레이아웃을 정의할때 주로 사용한다.
  • ID값은 언더바(_)를 사용하지않고 의미있는 단어로만 사용한다.(단어조합 불가능)
  • 레이아웃 컨텐츠 외 다른 내용들은 ID값을 사용하지않고 CLASS값으로 사용한다.
  • 독립적인 컨텐츠 또는 자바스크립트와 연동할 경우 ID값을 우선으로 사용한다.
  • CLASS값은 중복으로 여러개 정의가 가능하다.
  • CLASS값은 의미_구분_타입_순서 로 작성한다.
ID , CLASS 네이밍 예제
잘못된 예 올바른 예

.brand01_visual_history .brand_history_visual_type_01

1-3-3. 이미지

  • 이미지 예약어는 예약어표에 근거하여 작성한다.
  • 영문 소문자만을 사용하여 작성한다.
  • 같은 분류의 이미지가 두개 이상이면 마지막에 언더파(_)와 숫자를 추가하여 구분한다.
  • 특수문자는 사용할 수 없다.
  • 이미지 네이밍은 확장자와 상관없이 숫자로 작성한다.
  • 컨텐츠 이미지 경우 의미_구분_숫자 로 작성한다.
  • 공통으로 사용되는 이미지 경우 형태_의미_숫자 로 작성한다.

1-5. 기본 컨텐츠

  • 컨텐츠별 의미있는 클래스를 지정하여 최상위에 정의하여 공통사용하는 컨텐츠들도 상속받아 재사용이 가능하게 한다.
  • 컨텐츠별 단독 스크립트는 페이지 하단에 작성하거나 불러온다.
  • 컨텐츠별 내용은 최대한 이미지 사용을 배제한다.(디자인에 따른 케이스 바이 케이스)

A. 기본규칙

  • W3c Validation을 이용해 html5코드 내용을 테스트 한다.(https://validator.w3.org/)
  • 모든 태그의 속성은 큰따옴표("")로 감싼다
  • 들여쓰기를 이용하여 코드의 가독성을 높인다.
  • 3번의 줄바꿈으로 컨텐츠를 구분하고 1번의 줄바꿈으로 상세 컨텐츠를 구분한다.
<!-- HTML --> <section class="cell"> <h2>1. HTML</h2> <div class="small_content"></div> </section> <!-- //HTML --> <!-- CSS --> <section class="cell"> <h2>2. CSS</h2> <div class="small_content"></div> </section> <!-- //CSS -->

B. 링크

  • title 속성에 적절한 의미를 정의한다.
  • href 속성에 공백을 사용하지 않는다.
  • target 속성을 지정하여 링크의 흐름을 정확히 정의한다.
  • 페이지 전환이 아닌 UI용으로 작업되는 버튼은 a태그가 아닌 button태그로 작업을 지향한다.
<a href="#" title="확인" target="_self">확인</a>

C. 이미지

  • alt 속성을 내용이 없더라도 빈공간이라도 꼭 정의한다.
    • <img src="" alt=""/>

D. 테이블

  • caption 태그를 사용하여 무엇에 사용되는 테이블인지 반드시 정의한다.
  • tbody 와 thead를 정확히 구분하여 의미있게 컨텐츠를 구성한다.
  • table안에 table을 이용하는 컨텐츠는 최대한 배제한다.
  • colgroup을 이용하여 데이타의 넓이를 정의한다.
<table> <caption>샘플 테이블 표</caption> <colgroup> <col style="width:50%"> <col style="width:50%"> </colgroup> <thead> <tr> <th>타이틀 1</th> <th>타이틀 2</th> </tr> </thead> <tbody> <tr> <td>데이타 1</td> <td>데이타 2</td> </tr> </tbody> </table>

E. 입력

  • fieldset과 legend 를 이용하여 입력영역 간 구분하고 의미를 정의한다.
  • select선택 태그는 title에 의미를 정의한다.
  • input태그의 id값은 중복될 수 없다.
  • fieldset안의 table의 caption과 legend의 내용은 다르게 정의한다.
  • 휴대전화 , 이메일 등 각각 그룹핑 하여 작업하여 공통으로 사용할 수 있게 정의한다.
  • 한개의 form에 여러개의 fieldset이 사용될 수 있지만 한개의fieldset에 여러개의 form을 사용하지 않는다.
<fieldset> <legend>로그인 입력 폼</legend> <table> <caption>로그인 입력 테이블</caption> <colgroup> <col style="width:30%"> <col style="width:70%"> </colgroup> <tbody> <tr> <th>입력타이틀 1</th> <td><input type="text" name="name" id="name"></td> </tr> </tbody> </table> </fieldset>

2. CSS

2-1. 기본

A. 규칙

  • css2와 css3를 기준으로 작성한다.
  • 공백이 포함된 폰트명, 한글폰트명, 문자열 데이터타입, URL 데이터타입,filter속성의 파라미터값은 작은따옴표('')로 사용한다.
  • 프로젝트에 따른 CSS 작성 규칙에 맞게 정의한다.(한줄타입 , 여러줄 타입)
  • 그룹화 된 css 내용에는 반드시 주석으로 설명을 적어놓는다.
  • 3개의 빈줄은 컨텐츠별로 구분하고 1개의 빈줄은 세부요소별로 구분한다.
  • 그룹화 및 컨텐츠별 css 작성시 반드시 주석으로 해당 설명 정보를 작성한다.
  • hack css는 사용하지 않는다.
  • transition속성은 모든컨텐츠에 미리 사용하지않고 필요한 곳에서만 사용한다.
  • 독립적으로 사용될 css는 페이지내 inline style 로 작업한다.
    <input type="text" name="name" id="name" style="width:150px"/>
  • 최상단에 반드시 css 인코딩을 작성한다. 파일을 저장할때도 반드시 동일한 인코딩을 선택한다.
    @charset "utf-8";
  • css선언 타입은 External type을 기본으로 사용한다.
    <link rel="stylesheet" type="text/css" href="" />

2-2. 구조

  • common/css/폴더위치를 사용한다.
  • 웹폰트는 css내 import 방식이 아닌 html link태그를 이용하여 로드한다.
  • html의 link태그 사용시 rel , type , media 속성은 반드시 넣어준다.

2-3. 선택자

  • 공통 선택자 *는 페이지 성능을 떨어뜨림으로 사용하지 않는다.(부트스트랩 사용시 예외)
  • 여러조건을 만족시키는 속성구분(,)시에는 빈줄 한칸을 이용한다.
    선택자 속성구분 예제 테이블

    .bg01,.bg02{...} .bg01,
    .bg02{...}
  • ID네이밍은 하나의 단어로만 가능하며 class네이밍은 영문소문자 언더바(_)조합으로만 사용한다.(독립적인 페이지 작업시 대문자 작업 가능)
    css 네이밍 예제 테이블
    잘못된 예 올바른 예
    #brand_wrap #brand
    .brand-history-1 .brand_hisotry_01
  • 하위 선택자 작성시 반드시 하위선택자가 아닌 자식 선택자로 사용한다.
    하위 선택자 작성 예제 테이블
    잘못된 예 올바른 예
    #gnb ul li #gnb > ul > li

    하위선택자의 속성 설정 예제 테이블
    잘못된 예 올바른 예
    #brand .brand_wrap .brand01 .obj01 img{...} .brand01 .obj01 img{...}
  • ID값으로 하위선택자 속성 설정시 아이디와 아이디로 설정하지 않는다.
    ID값으로 하위선택자 설정 예제 테이블
    잘못된 예 올바른 예

  • 태그.클래스 작성방식으로 사용되는 컨텐츠는 공통컨텐츠에서만 사용한다.

2-4. 속성

  • 속성 설정시 마지막 속성의 세미콜론(;)은 삭제한다.
  • 속성 순서는 아래와 같은 방법을 기본으로 작성한다.
    • { position , display , width height , font , margin , padding , border , background , etc... }

A. background

  • 여러 이미지를 처리할 경우에만 background속성을 나뉘어 적용하고 기본적으로는 한번에 설정한다.

B. border

  • border box의 속성은 기본으로 content-box로 설정되어 사용한다.
  • border의 다중 컨트롤시 시계방향을 기준으로 작성한다.(top,right,bottom,left)

C. clear

  • float:left 되는 부모 객체에 overflow:hidden 방식을 기본으로 한다.
  • 표현의 방향을 고려하여 after선택자를 사용해도된다.(clear:both,content:'';display:block)

D. margin

  • 컨텐츠별 폰트 간격 및 정렬시 margin-top값을 기준으로 작성한다.
  • margin-bottom값은 컨텐츠별 타이틀 영역에 사용한다.

D. z-index

  • header,footer,pop 등 레이아웃의 범위에 벗어나지않는 한에서 z-index를 설정한다.

D. position

  • fixed 속성을 가진 객체의 하위객체는 fixed속성을 다시 가질 수 없다.

4. MOBILE

4-1. 기본

  • 해상도별(min:320 ~ max 414)사이즈에 맞게 미디어쿼리를 이용하여 작업한다.
  • meta 태그의 viewport의 넓이를 디바이스 넓이로 설정하여 작업한다.
    <meta name="viewport" content="width=device-width">
  • iframe태그는 특별한 이슈가 없는한 배제한다.(youtube 제외)
  • 스크롤바 스크립트 라이브러리는 최대한 사용하지 않고 css속성의 overflow scroll속성을 사용한다.
  • 가로로 긴 컨텐츠의 경우 모바일 해상도에 맞게 디자인이 되어있지 않은 컨텐츠는 협업후에 가로 스크롤을 적용한다.
  • 폰트의 경우 시스템 폰트를 사용하는걸로 위주로 하고 크기단위를 px 아닌 rem 으로 지향한다.

4-2. 이미지

  • 실제 물리해상도의 디자인 작업 크기에 2배를 키워서 작업한다.
  • 스프라이트 사용 또는 일반 사용시 이미지 크기단위는 짝수로 떨어지게 작업한다.(2배수,1배수 모두)
  • 이미지 태그를 이용하여 페이지내에 존재할 경우 width 값을 반드시 설정한다.
    <img src="" alt="" width="20" />
  • 배경으로 이미지가 작업될 경우 background-size 속성을 이용한다.
    -webkit-background-size:20px auto; background-size:20px auto

4-3. css

  • css3를 기준으로 작성한다.
  • 미디어 쿼리를 이용해 모바일 디바이스별 물리해상도에 대응하여 작업한다.
    @media screen and (max-width:414px){ .셀렉터{속성} }
  • css3로 표현할수 있는 최대한으로 표현하며 이미지 사용을 지양한다.
  • 폰트사이즈의 단위는 em 또는 px로 작업한다.

5. SEO & SNS

5-1. 검색엔진최적화(SEO)

  • 각 페이지별 title 과 description 의 내용을 의미에 맞는 페이지 타이틀과 정보로 작성한다.(인클루드시 개발연동)
    <title>이벤트-함께하면 좋은 이벤트</title> <meta name="title" content="이벤트-함께하면 좋은 이벤트"> <meta name="Description" content="이벤트 당첨시 100만원 획듯!"/>
  • link 태그의 canonical 대표URL을 등록하여 혼란을 방지한다.
    <link rel="canonical" href="대표URL">
  • 컨텐츠내용각각의 제목 태그를 적절하게 사용한다.
    <section id="container"> <h2 class="page_title">마이페이지</h2> <p class="page_info"></p> <div class="mypage_main"> <h3>메인</h3> </div> </section>
  • 개발에 따라 다르겟지만 기본적으로 알기 쉬운 URL을 사용한다.
    /product/list.html(X) /product/best_list.html(O) /product/product_list.html(O)
  • xml사이트맵을 작성하여 최상단 루트에 작성한다.
  • 페이지내에 링크로 이루어지는 컨텐츠들은 이미지사용을 지양한다.
  • robots.txt파일을 작성하여 활용한다.
  • 백그라운드 이미지를 사용시 정보 텍스트를 작성한다.

5-2. 소셜네트워크(SNS)

  • 페이지별 대표이미지가 다르거나 사이트정보를 다르게 할 경우 개발자와 협업하여 작업한다.
  • meta 태그의 og속성을 이용하여 사이트 정보 및 대표이미지를 구성한다.
